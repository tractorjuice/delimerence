## Status

**DRAFT**

## Context

DeLimerence must assess each user's current limerence phase to tailor its intervention strategy. Limerence progresses through four phases (Tennov, 1979):

1. **Initiation** — early attraction, beginning of intrusive thoughts about the Limerent Object (LO)
2. **Crystallisation** — peak idealisation, maximum obsessive thinking consuming up to 85% of waking hours, compulsive rituals at their most intense
3. **Deterioration** — the idealised fantasy crumbling, grief emerging, disillusionment setting in
4. **Resolution** — limerence fading, return to emotional baseline

Phase assessment is clinically critical because the appropriate intervention differs radically by phase. A user in crystallisation needs aggressive reality-testing — naming the dopamine reward prediction error, confronting the idealisation with evidence [LM35-C12]. A user in deterioration needs grief support and relapse prevention — reality-testing someone who is already grieving the loss of the fantasy is cruel and counterproductive. Misassessment is not merely a poor user experience; it is a direct harm vector [R-008].

The MHRA regulatory positioning established in ADR-004 requires that phase assessment be framed as *response calibration* — adjusting the tool's conversational strategy — rather than clinical diagnosis. The system determines "which kind of support to provide", not "which condition you have". This framing is essential to maintaining the tool's classification as a general wellness product rather than a medical device.

This decision addresses FR-003 (Phase Assessment Engine), FR-005 (Psychoeducation Delivery — phase determines which education to deliver), FR-007 (Anti-Warmth — phase determines intervention intensity), and BR-001 (Deliver Recovery Tool). It directly mitigates R-008 (Phase misassessment causes user harm, residual score 8).

## Decision Drivers

- **Clinical safety obligation**: Delivering the wrong intervention for the user's limerence phase causes direct psychological harm — aggressive reality-testing during deterioration compounds grief; grief support during crystallisation reinforces the fantasy [R-008]
- **Regulatory framing constraint**: ADR-004 establishes that DeLimerence must avoid medical device classification; phase assessment must be positioned as response calibration, not diagnosis [FR-003 acceptance criteria, NFR-C-003]
- **Natural conversation requirement**: Users in acute limerent distress are not in a state to complete clinical questionnaires; assessment must feel like conversation, not interrogation [BR-001 rationale: available at 3am, at the point of crisis]
- **Reliable data extraction**: Phase determination must be machine-readable to drive downstream logic (response strategy selection, psychoeducation content selection, anti-warmth intensity calibration) — free-text LLM output is insufficient [FR-003, FR-005, FR-007]
- **Safe default requirement**: When assessment is uncertain or incomplete, the system must default to psychoeducation-only mode as the least-harmful response strategy [FR-003 acceptance criteria: inconclusive assessment defaults to psychoeducation-only]
- **Reassessment capability**: Limerence phases are not static; users may shift between phases across sessions or even within a session, and the system must detect and respond to phase transitions [FR-003 acceptance criteria: returning user phase shift detection]
- **Clinical governance**: The assessment protocol and question set must be reviewable and approvable by the clinical advisory board before deployment

## Considered Options

1. **Do Nothing** — No phase assessment, same response for all users
2. **LLM-Driven Assessment with Structured Output** (chosen)
3. **Structured Questionnaire (Rule-Based)**
4. **Hybrid: Questionnaire + LLM Interpretation**

## Decision Outcome

**Chosen option: "LLM-Driven Assessment with Structured Output"**, because it is the only option that provides natural conversational assessment integrated into the recovery dialogue, reliable machine-readable phase determination via structured output (tool use / function calling), and a clinically reviewable protocol — while avoiding the MHRA risk of a structured diagnostic questionnaire that could be interpreted as a medical device component.

### Y-Statement

In the context of tailoring limerence interventions by phase, facing the risk that misassessment could cause user harm, we decided for LLM-driven phase assessment with structured output validation, to achieve natural conversational assessment that integrates seamlessly with the recovery dialogue, accepting the need for clinical advisory board protocol approval and ongoing accuracy monitoring.

### Consequences

**Positive:**

- Natural conversational flow during onboarding — users in acute distress are met with dialogue, not a clinical intake form, reducing abandonment risk [BR-001]
- Structured output (tool use / function calling) ensures phase determination is reliably extracted as a discrete value (`initiation`, `crystallisation`, `deterioration`, `resolution`) rather than parsed from free text — eliminating a class of parsing failures
- Phase determination drives downstream systems deterministically: response strategy selection [FR-003], psychoeducation content selection [FR-005], and anti-warmth intensity calibration [FR-007]
- Conversational assessment is harder for MHRA to classify as a diagnostic instrument than a structured questionnaire with scored items — supports ADR-004 regulatory positioning
- Clinical advisory board can review and approve the assessment protocol (system prompt instructions, question guidelines, phase determination criteria) as a governed artefact
- Reassessment is natural — the LLM can detect phase indicators mid-conversation and trigger re-evaluation without interrupting the dialogue flow
- Safe default is architecturally enforced — when the LLM returns low confidence or an inconclusive assessment, the system defaults to psychoeducation-only mode [FR-003 acceptance criteria]

**Negative:**

- LLM assessment accuracy is unvalidated — no ground truth dataset exists for limerence phase classification; accuracy depends on the quality of the assessment protocol and the LLM's ability to interpret self-reported emotional states
- Depends entirely on user self-report — users in crystallisation may lack insight into their own state (the idealisation that defines crystallisation impairs self-assessment) [Tennov, 1979]
- Clinical advisory board must exist and approve the protocol before this feature can be deployed — the board is not yet constituted [ARC-001-RISK-v1.0 Key Findings]
- Structured output adds a dependency on the LLM provider's tool use / function calling API — if the provider changes this interface, the integration breaks [ADR-001 dependency]
- No reproducibility guarantee — the same user responses may yield different phase assessments on different invocations due to LLM non-determinism; mitigation requires confidence thresholds and conservative defaults
- Ongoing accuracy monitoring requires a feedback mechanism (clinical advisory board case review, user-reported "this doesn't feel right" reassessment triggers) that adds operational overhead

### Confirmation

The decision will be validated through:

- Clinical advisory board review and sign-off on the phase assessment protocol, question guidelines, and phase determination criteria before deployment
- Validation of phase assessment accuracy against clinical judgement — target > 80% agreement between LLM-determined phase and clinician-assessed phase for a sample of onboarding transcripts [ARC-001-RISK-v1.0, Action 12]
- User feedback mechanism: "this doesn't feel right" trigger causes immediate reassessment and defaults to psychoeducation-only mode until reassessment completes [R-008 response strategy]
- Monitoring of phase distribution across the user population — clinical advisory board reviews aggregate phase distribution quarterly to detect systematic bias (e.g., over-classification of crystallisation)
- Structured output validation: 100% of phase assessments must return a valid phase enum value; any structured output parsing failure triggers the safe default (psychoeducation-only)
- Red-team testing with adversarial user inputs designed to manipulate phase assessment (e.g., user deliberately describing crystallisation symptoms while in resolution to obtain different responses)

## Pros and Cons of the Options

### Option 1: Do Nothing — No Phase Assessment

**Description**: Deliver the same response strategy to all users regardless of their limerence phase. No assessment, no tailoring, no phase-aware behaviour.

- Good, because zero implementation complexity — no assessment logic, no structured output, no clinical protocol to approve
- Good, because no risk of misassessment — cannot misassess what is not assessed
- Bad, because cannot tailor interventions to the user's phase, violating the core clinical premise that different phases require different support [FR-003]
- Bad, because reality-testing a user in deterioration phase could compound grief and cause direct harm [R-008]
- Bad, because grief support for a user in crystallisation phase reinforces the fantasy rather than challenging it
- Bad, because psychoeducation is generic rather than contextual — explaining dopamine reward prediction error is most impactful during crystallisation when the user is experiencing it [FR-005]
- Bad, because fails to meet FR-003 acceptance criteria entirely
- Bad, because undermines the clinical advisory board's ability to approve phase-appropriate intervention protocols

### Option 2: LLM-Driven Assessment with Structured Output (Chosen)

**Description**: Use the LLM (Claude via AWS Bedrock, per ADR-001) to conduct phase assessment through structured conversational questions during onboarding. The LLM follows a clinical advisory board-approved assessment protocol embedded in the system prompt. Phase determination is extracted via structured output (tool use / function calling) returning one of four phase values plus a confidence indicator. The determined phase is stored in the user profile and drives response strategy selection, psychoeducation content delivery, and anti-warmth intensity calibration. Re-assessment is triggered when conversational indicators suggest a phase shift or when the user reports that the current approach "doesn't feel right".

- Good, because natural conversational flow reduces onboarding abandonment — users in distress are met with dialogue, not a form
- Good, because structured output ensures reliable, machine-readable phase determination without free-text parsing
- Good, because the LLM can adapt follow-up questions based on the user's language and emotional state, probing ambiguous indicators conversationally
- Good, because conversational framing supports ADR-004 regulatory positioning — response calibration, not diagnosis
- Good, because reassessment is seamless — phase shift indicators detected mid-conversation trigger re-evaluation without interrupting the session
- Good, because clinical advisory board can review and version-control the assessment protocol as a governed artefact
- Neutral, because accuracy is unvalidated and requires ongoing monitoring — no existing ground truth for limerence phase classification
- Bad, because depends on user self-report, which is unreliable during crystallisation (impaired insight is a feature of the phase)
- Bad, because LLM non-determinism means the same inputs may produce different assessments — mitigated by confidence thresholds and conservative defaults
- Bad, because clinical advisory board must approve the protocol before deployment, and the board is not yet constituted

### Option 3: Structured Questionnaire (Rule-Based)

**Description**: Present a fixed questionnaire with scored items (e.g., "How many hours per day do you spend thinking about this person?", "Do you feel the relationship is ending?") and map aggregate scores to phases algorithmically. No LLM involvement in the assessment itself.

- Good, because deterministic and reproducible — the same answers always yield the same phase
- Good, because easier to validate clinically — fixed items can be tested against clinician assessments in a controlled study
- Good, because no LLM dependency for the assessment component — works even if the LLM provider changes
- Good, because simpler to audit — the scoring algorithm is transparent and version-controlled
- Bad, because rigid — does not adapt to the user's language, emotional state, or cultural context
- Bad, because feels clinical and impersonal — presenting a scored questionnaire to someone in acute distress at 3am reduces engagement and increases abandonment [BR-001]
- Bad, because harder to reassess mid-session — requires interrupting the conversation to re-administer the questionnaire
- Bad, because MHRA may interpret a structured diagnostic questionnaire with scored items as more "device-like" than a conversational assessment, increasing medical device classification risk [ADR-004, R-002]
- Bad, because fixed items cannot probe ambiguous indicators — a user who scores between phases receives no follow-up clarification

### Option 4: Hybrid — Questionnaire + LLM Interpretation

**Description**: Use structured questions for data collection (ensuring consistent input), then pass the responses to the LLM for phase interpretation and conversational delivery. The questionnaire provides reliable data; the LLM provides natural conversation and nuanced interpretation.

- Good, because combines the reliability of structured input with the natural conversation of LLM delivery
- Good, because structured data collection ensures all necessary indicators are captured
- Good, because LLM interpretation can handle ambiguous cases that rule-based scoring cannot
- Bad, because more complex architecture — two systems (questionnaire engine + LLM interpreter) to build, maintain, and validate
- Bad, because clinical validation is needed for both the questionnaire design and the LLM interpretation — doubling the clinical advisory board workload
- Bad, because the structured questionnaire component still carries MHRA classification risk [ADR-004]
- Bad, because user experience is disjointed — transition from questionnaire to conversation creates a seam that undermines the "tool that talks to you" design intent
- Bad, because reassessment still requires re-administering the questionnaire portion, losing the seamless mid-conversation detection advantage
