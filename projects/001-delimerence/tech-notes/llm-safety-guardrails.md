## Summary

LLM safety guardrails are software layers that enforce quality, safety, and behavioural constraints on LLM inputs and outputs before they reach end users. As of April 2026, the ecosystem has matured significantly with two dominant open-source frameworks (NVIDIA NeMo Guardrails and Guardrails AI) and emerging commercial offerings. For safety-critical applications like mental health tools, the recommended architecture is defence-in-depth: combining conversation flow control (NeMo), output validation (Guardrails AI), and domain-specific custom classifiers.

## Key Findings

- **NVIDIA NeMo Guardrails** (Apache 2.0, 5,600+ GitHub stars) is the most mature framework for conversational AI guardrails. It controls conversation flow, restricts topics, and handles unsafe inputs. The 2026 IORails engine supports parallel execution of multiple guardrails with 50-200ms latency. Production-ready container image available for Kubernetes deployment via Helm charts. Supports OpenAI, Anthropic, HuggingFace, and NVIDIA NIM models. OpenAI-compatible API endpoint and LangChain integration via GuardrailsMiddleware.

- **Guardrails AI** (Apache 2.0) provides output-focused validation with 50+ pre-built validators from the Guardrails Hub (PII detection, toxicity, hallucination). Composable validator architecture with configurable failure actions (correction, retry, filtering). Lighter weight than NeMo -- runs in-process with minimal latency overhead. Best used as a complementary layer to NeMo's flow control.

- **Mental health-specific guardrails** are an emerging area. A 2026 Nature study evaluated the Verily Mental Health Guardrail (VMHG) on clinician-labelled datasets containing crisis messages, demonstrating the critical need for specialised safety mechanisms in mental health applications. Standard guardrails (toxicity, PII) are necessary but insufficient for clinical applications.

- **Defence-in-depth architecture** is the consensus best practice: no single guardrail catches everything, but multiple layers dramatically reduce the probability of failures reaching users. The recommended stack is: pre-call input validation (topic control, jailbreak detection), during-call streaming moderation, and post-call response filtering (content safety, domain-specific checks).

- **Latency impact** of guardrails is typically 50-200ms per request for NeMo, and negligible for in-process Guardrails AI validators. This is within acceptable bounds for most conversational applications (total budget typically 3-10 seconds).

- **Custom classifiers** are still required for domain-specific safety needs. For DeLimerence, this means building a warmth-detection classifier and diagnostic-claim filter that no off-the-shelf guardrail provides.
